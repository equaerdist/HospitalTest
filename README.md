# Doctor Management API

Этот проект является недоделанным и содержит базовую реализацию функционала по работе с докторами. Тем не менее, он предоставляет достаточный объем функционала, чтобы продемонстрировать основной механизм работы приложения.

## Что реализовано

- CRUD операции для сущности **Doctor**
- Простая модель манипуляции данными через Entity Framework Core
- Основные контроллеры для взаимодействия с API

## Что еще можно улучшить

- **Глобальный Middleware для обработки исключений**: Сейчас ошибки обрабатываются локально, но добавление глобального Middleware позволит унифицировать обработку исключений и улучшить логирование.
- **Реализация сервисов репозиториев**: В текущем состоянии данные манипулируются напрямую через DbContext. В будущем можно добавить сервисы репозиториев для лучшего управления данными и инкапсуляции логики работы с базой данных.
- **Расширение функционала**: Приложение можно расширить, добавив больше бизнес-логики и новых возможностей для управления другими сущностями.

## Как начать

1. **Настройка конфигурации**: Все чувствительные данные хранятся в классе `AppConfig`. Чтобы они корректно забиндились, необходимо указать их в секретах приложения или в файле конфигурации.

2. **Запуск приложения**: Склонируйте репозиторий и запустите приложение. Убедитесь, что все необходимые данные конфигурации указаны, и база данных настроена правильно.
